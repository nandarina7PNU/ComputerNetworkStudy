# 1.1 인터넷이란 무엇인가?

인터넷은 전 세계의 수많은 컴퓨팅 장치들을 서로 연결하는 거대한 컴퓨터 네트워크이다. 이 장에서는 인터넷을 두 가지 관점에서 설명한다.

1. **구성 요소 중심 관점**
2. **서비스 제공 인프라 관점**

---

## 1. 구성 요소 중심의 인터넷

인터넷은 수십억 개의 장치들이 서로 연결된 네트워크이다. 여기서 인터넷에 연결된 장치들은 **호스트(host)** 또는 **종단 시스템(end system)** 이라고 부른다.

대표적인 종단 시스템은 다음과 같다.

* 데스크톱 컴퓨터
* 노트북
* 스마트폰
* 태블릿
* 서버
* TV, 자동차, 온도 조절 장치, 가전제품 같은 IoT 장치

즉, 오늘날의 인터넷은 단순히 컴퓨터만 연결하는 네트워크가 아니라, 다양한 사물까지 연결하는 거대한 시스템이다.

---

## 2. 패킷과 패킷 스위치

인터넷에서 데이터는 한 덩어리로 그대로 전송되지 않는다. 송신 측 종단 시스템은 데이터를 여러 조각으로 나누고, 각 조각에 헤더 정보를 붙인다.

이렇게 만들어진 데이터 조각을 **패킷(packet)** 이라고 한다.

패킷은 네트워크를 지나 목적지까지 이동한 뒤, 수신 측 종단 시스템에서 다시 원래 데이터로 재조립된다.

패킷을 전달하는 장비를 **패킷 스위치(packet switch)** 라고 한다. 대표적인 패킷 스위치는 다음 두 가지이다.

* **라우터(router)**
* **링크 계층 스위치(link-layer switch)**

라우터와 스위치는 패킷을 최종 목적지 방향으로 전달한다.

패킷이 송신자에서 수신자까지 이동하면서 거치는 통신 링크와 패킷 스위치들의 순서를 **경로(route)** 또는 **패스(path)** 라고 한다.

---

## 3. 패킷 교환 네트워크 비유

패킷 교환 네트워크는 교통망에 비유할 수 있다.

| 네트워크 구성 요소 | 교통망 비유   |
| ---------- | -------- |
| 패킷         | 트럭       |
| 통신 링크      | 도로, 고속도로 |
| 패킷 스위치     | 교차로      |
| 종단 시스템     | 건물       |

공장에서 목적지 창고로 화물을 보낼 때, 화물을 여러 트럭에 나누어 싣고 각각의 트럭이 도로를 따라 이동한다. 마찬가지로 인터넷에서는 데이터를 여러 패킷으로 나누고, 각 패킷이 네트워크를 따라 목적지까지 이동한다.

---

## 4. ISP와 인터넷의 구조

종단 시스템은 보통 **ISP(Internet Service Provider)** 를 통해 인터넷에 접속한다.

ISP의 예시는 다음과 같다.

* 가정용 인터넷 제공자
* 기업 ISP
* 대학 ISP
* 공항, 호텔, 카페의 WiFi 제공자
* 이동통신사의 셀룰러 데이터망

각 ISP 자체도 패킷 스위치와 통신 링크로 이루어진 하나의 네트워크이다.

인터넷은 하나의 단일 네트워크가 아니라, 여러 ISP 네트워크들이 서로 연결된 **네트워크들의 네트워크(network of networks)** 이다.

하위 ISP들은 상위 ISP와 연결되고, 상위 ISP들은 서로 직접 연결된다. 이를 통해 전 세계의 종단 시스템들이 서로 통신할 수 있다.

---

## 5. TCP/IP와 인터넷 표준

인터넷의 구성 요소들은 데이터를 주고받기 위해 **프로토콜(protocol)** 을 실행한다.

인터넷에서 가장 중요한 프로토콜은 다음 두 가지이다.

* **TCP(Transmission Control Protocol)**
* **IP(Internet Protocol)**

특히 IP는 라우터와 종단 시스템 사이에서 주고받는 패킷의 형식을 정의한다. 인터넷의 주요 프로토콜 집합을 통칭하여 **TCP/IP** 라고 부른다.

프로토콜은 서로 다른 시스템들이 호환되게 동작하기 위해 표준화되어야 한다. 인터넷 표준은 주로 **IETF(Internet Engineering Task Force)** 에 의해 개발된다.

IETF의 표준 문서는 **RFC(Requests for Comments)** 라고 부른다. RFC는 TCP, IP, HTTP, SMTP 같은 주요 인터넷 프로토콜을 정의한다.

---

## 6. 서비스 제공 인프라로서의 인터넷

인터넷은 단순히 장비들의 집합이 아니다. 인터넷은 애플리케이션에게 서비스를 제공하는 인프라이기도 하다.

인터넷 위에서 동작하는 애플리케이션에는 다음과 같은 것들이 있다.

* 이메일
* 웹 브라우징
* 메신저
* 지도 서비스
* 음악 스트리밍
* 영화 및 TV 스트리밍
* 소셜 미디어
* 화상 회의
* 온라인 게임
* 위치 기반 추천 서비스

이러한 애플리케이션들은 여러 종단 시스템이 서로 데이터를 교환하며 동작하므로 **분산 애플리케이션(distributed application)** 이라고 부른다.

중요한 점은 애플리케이션이 네트워크 핵심부의 라우터나 스위치에서 실행되는 것이 아니라, **종단 시스템에서 실행된다**는 것이다.

라우터와 스위치는 애플리케이션의 내용을 이해하지 않고, 단지 데이터를 목적지 방향으로 전달하는 역할을 한다.

---

## 7. 소켓 인터페이스

분산 애플리케이션을 만들려면 서로 다른 종단 시스템에서 실행되는 프로그램들이 데이터를 주고받아야 한다.

이때 프로그램은 인터넷에게 데이터를 전달해 달라고 요청해야 한다. 이를 위해 종단 시스템은 **소켓 인터페이스(socket interface)** 를 제공한다.

소켓 인터페이스는 프로그램이 인터넷을 통해 다른 프로그램에게 데이터를 보내기 위해 따라야 하는 규칙들의 집합이다.

우편 서비스를 이용할 때 편지를 봉투에 넣고, 주소를 쓰고, 우표를 붙인 뒤, 우체통에 넣어야 하는 것처럼, 인터넷에서도 데이터를 보내는 프로그램은 정해진 인터페이스를 따라야 한다.

---

## 8. 프로토콜이란 무엇인가?

프로토콜은 통신을 수행하기 위한 규칙이다.

사람 사이의 대화에서도 프로토콜이 존재한다. 예를 들어 시간을 물어볼 때 보통 다음과 같은 순서를 따른다.

1. 인사를 한다.
2. 상대방이 응답한다.
3. 질문을 한다.
4. 상대방이 대답한다.

이처럼 통신에는 정해진 메시지 형식과 순서, 그리고 각 메시지를 받았을 때의 행동이 있다.

네트워크에서도 마찬가지이다. 컴퓨터, 스마트폰, 서버, 라우터 같은 장치들은 프로토콜에 따라 메시지를 주고받고 동작한다.

---

## 9. 프로토콜의 정의

프로토콜은 다음을 정의한다.

* 통신 개체들이 주고받는 메시지의 형식
* 메시지가 교환되는 순서
* 메시지를 보내거나 받았을 때 수행해야 하는 동작
* 특정 사건이 발생했을 때 수행해야 하는 동작

정리하면 다음과 같다.

> 프로토콜은 둘 이상의 통신 개체 사이에서 교환되는 메시지의 형식과 순서, 그리고 메시지의 송수신 또는 다른 사건이 발생했을 때 수행할 행동을 정의한다.

---

## 10. 웹 요청 예시

웹 브라우저에 URL을 입력하면 다음과 같은 일이 일어난다.

1. 사용자의 컴퓨터가 웹 서버에 연결 요청 메시지를 보낸다.
2. 웹 서버가 연결 응답 메시지를 보낸다.
3. 사용자의 컴퓨터가 원하는 웹 페이지를 요청하는 GET 메시지를 보낸다.
4. 웹 서버가 해당 웹 페이지 파일을 사용자 컴퓨터로 보낸다.

이 과정 전체도 정해진 프로토콜에 따라 이루어진다.

---

## 핵심 정리

* 인터넷은 수많은 종단 시스템, 통신 링크, 패킷 스위치, ISP가 연결된 거대한 네트워크이다.
* 인터넷에 연결된 장치들을 호스트 또는 종단 시스템이라고 한다.
* 데이터는 패킷 단위로 나뉘어 전송된다.
* 라우터와 링크 계층 스위치는 패킷을 목적지 방향으로 전달한다.
* 인터넷은 여러 ISP 네트워크가 연결된 네트워크들의 네트워크이다.
* 인터넷은 애플리케이션에게 통신 서비스를 제공하는 인프라이다.
* 인터넷 애플리케이션은 종단 시스템에서 실행된다.
* 소켓 인터페이스는 프로그램이 인터넷을 통해 데이터를 보내기 위해 사용하는 규칙이다.
* 프로토콜은 메시지의 형식, 순서, 그리고 송수신 시 동작을 정의한다.
* 컴퓨터 네트워킹을 이해한다는 것은 결국 다양한 네트워크 프로토콜의 역할, 이유, 동작 방식을 이해하는 것이다.
